## 2024-05-18 - Added keyboard accessibility to presentation deck
**Learning:** The presentation deck had a `#present-btn` that only indicated its action via a FontAwesome icon (`<i class="fa-solid fa-desktop"></i>`). While there was text "Present", screen readers could get confused by the icon without `aria-hidden="true"`. Furthermore, when switching to presentation mode, there was no shortcut hint. Adding `aria-label` and `title` to the button and dynamically updating them via JavaScript improves both screen reader experience and regular mouse/keyboard accessibility by exposing the `(Esc)` hint. Also added `:focus-visible` to interactive elements to help keyboard navigators, matching the color theme (`#B89E6B`).
**Action:** Always add `aria-hidden="true"` to pure decorative FontAwesome icons, add `aria-label`/`title` tooltips for action buttons, and remember to update `aria-label` state via JavaScript for toggle buttons. Ensure `:focus-visible` outlines match the design system's accent color.
